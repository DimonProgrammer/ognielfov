## 1. КОНТЕКСТ ПРОЕКТА

### Компания и задача
Российский производитель мерча и кастомной печати. Технологии: DTF, шелкография, сублимация, УФ-печать, вышивка, тампопечать. Задача: создать мультисегментный сайт, который одновременно работает как инструмент лидогенерации для трёх ЦА и как SEO-машина в нише с низкой конкуренцией по нишевым запросам.

### Стек
| Компонент | Решение |
|-----------|---------|
| Фреймворк | Next.js (ISR для каталога) **или** Astro (hybrid mode) — решается в ответе на вопрос #1 |
| CSS | Tailwind CSS |
| UI-компоненты | shadcn/ui |
| Уведомления | Telegram Bot API (заявки из форм → чат менеджера) |
| Хостинг | Vercel (рекомендуется для Next.js) или VPS |
| SEO | Schema.org JSON-LD, sitemap.xml, robots.txt, Яндекс.Вебмастер |

### Сроки
| Вариант | Срок | Условие |
|---------|------|---------|
| MVP-компромисс | 9–11 дней | Статический каталог (JSON), без CMS, без блога, минимум анимаций |
| Полноценный сайт | 17–24 дня | CMS (Sanity/Strapi), динамический каталог, блог, все сегменты |
| **Рекомендация** | **3–4 недели** | Один разработчик, все 12–14 шаблонов, адаптив, интеграции |

### 3 сегмента ЦА (по одной строке)
| # | Сегмент | Описание |
|---|---------|---------|
| 1 | **B2C** | Частники, команды, блогеры, организаторы ивентов — малые тиражи от 1–30 шт., решение принимают за 1–7 дней, средний чек 15–80 тыс. руб. |
| 2 | **WB/Ozon-продавцы** | ИП/ООО, строящие СТМ на маркетплейсах — нужны документы, тест-партии от 50 шт., стабильное качество, средний чек 50–800 тыс. руб. |
| 3 | **Корпоративные клиенты** | HR, маркетологи, event-менеджеры компаний от 50 чел. — нужны гарантия сроков, НДС, брендбук, средний чек 150 тыс. – 2 млн руб. |

## 3. ПОЗИЦИОНИРОВАНИЕ: НЕЗАНЯТАЯ НИША

### Что делают ВСЕ конкуренты (не повторять)
- Заявляют «качество», «опыт», «профессионализм» без доказательств
- Прячут цены у B2B: «оставьте заявку, мы перезвоним»
- Работают в 1–2 сегментах (либо только B2C, либо только корпоратив)
- Нет онлайн-калькулятора для B2B
- Нет образца перед тиражом как стандартной услуги (обещают, не гарантируют)
- Листинг услуг без объяснений: «делаем DTF, шелкографию, сублимацию» без «когда что»

### Что не делает никто (наши возможности)
| Возможность | Потенциал |
|-------------|-----------|
| Чёткое 3-оконное разделение по сегментам (B2C / WB-Ozon / Корпоратив) на одном сайте | Единственный охватывающий все 3 ниши — без прямых конкурентов |
| Онлайн-калькулятор цен для B2B (прозрачность) | Главный барьер рынка устранён — конверсия без менеджера |
| Страница «Для маркетплейсов»: полный цикл с маркировкой, упаковкой и отгрузкой на WB/Ozon | Голубой океан: конкуренты делают это частично, никто — полностью |
| Образовательный контент: «когда DTF, когда шелкография» | Клиент приходит образованным → меньше согласований → выше LTV |
| Гарантия сроков + образец перед тиражом как стандарт, а не исключение | Главная боль всех сегментов закрыта |

### Рекомендуемое УТП (1–2 предложения)
> **«Производим мерч без сюрпризов: фиксированные сроки, образец перед тиражом, прозрачные цены — для частных заказов от 1 штуки, продавцов маркетплейсов и корпоративных команд»**

### Позиционирование по сегментам
| Сегмент | Формулировка |
|---------|-------------|
| **B2C** | «Твоя идея — наш мерч. От 1 штуки, за 5 дней. Без минималки и китайских сюрпризов» |
| **WB/Ozon** | «Российское производство для маркетплейсов: тест-партия → масштаб → документы для WB под ключ» |
| **Корпоратив** | «Мерч, которым гордятся сотрудники. Персональный менеджер, гарантия сроков в договоре, НДС в комплекте» |

## 4. ЦЕЛЕВАЯ АУДИТОРИЯ: КЛЮЧЕВЫЕ ИНСАЙТЫ

### Сегмент 1: B2C (частные, команды, блогеры)

| Параметр | Данные |
|----------|--------|
| **Уровень осведомлённости (Шварц)** | Уровень 2–3: осознают потребность, ищут решение, сравнивают 3–5 сайтов |
| **Главная боль** | Непонимание процесса + страх: «отдам деньги, получу брак или вообще ничего» |
| **Главный страх перед заказом** | «Вдруг качество окажется ужасным, а деньги уже потрачены?» + «Не успеют к дате» |
| **Главный триггер к действию** | «От 1 штуки» / «От 10 штук» + реальные фото работ (не рендеры) + быстрый ответ в мессенджере |
| **Что должно быть на сайте** | Минимальный тираж без скролла, портфолио с реальными фото, раздел «Как это работает», ориентировочные цены или калькулятор, кнопка Telegram |
| **RAS-триггеры** | «от 1 штуки», «образец перед тиражом», «за 5 дней», «реальные фото работ», «ответим за 15 минут» |

### Сегмент 2: WB/Ozon-продавцы (СТМ)

| Параметр | Данные |
|----------|--------|
| **Уровень осведомлённости (Шварц)** | Уровень 3–4 (55%): активно ищут производителя, сравнивают, запрашивают образцы |
| **Главная боль** | Нестабильное качество от партии к партии + незнание производителем требований маркетплейса |
| **Главный страх перед заказом** | «Пришлют некачественную партию → получу негативные отзывы на WB → потеряю позиции» |
| **Главный триггер к действию** | Упоминание WB/Ozon + «от 50 штук» + «маркировка Честный знак» + кейсы селлеров |
| **Что должно быть на сайте** | Отдельная страница «Для маркетплейсов», кейсы с цифрами, тест-партия, условия дозаказа, контакт через Telegram |
| **RAS-триггеры** | «от 50 штук», «документы WB/Ozon», «Честный знак», «стабильное качество», «отгрузка на склад», «тест-партия» |

### Сегмент 3: Корпоративные клиенты

| Параметр | Данные |
|----------|--------|

| **Что должно быть на сайте** | Кейсы B2B с брендами, работа с юрлицами/НДС, персональный менеджер, портфолио корп. проектов |
| **RAS-триггеры** | «корпоративный мерч под ключ», «НДС», «гарантия сроков», «цветопроба», «личный менеджер», «кейсы [известные бренды]» |

## 5. HERO-СЕКЦИЯ: СТРАТЕГИЯ

### На каком уровне строить главный месседж
**Уровень 3 по Шварцу** — «Осведомлённые о решении». Человек знает, что хочет мерч, ищет производителя. Hero отвечает на вопрос «почему именно вы?», а не «что такое мерч?».

Дополнительно: Hero содержит разветвление по сегментам (3 CTA-кнопки) — каждый уходит на свой путь.

### Рекомендуемая структура Hero

| Элемент | Содержание |
|---------|-----------|
| **H1** | Ключевое ценностное утверждение (вариант на выбор — см. ниже) |
| **Подзаголовок** | «DTF, шелкография, вышивка — 6 технологий нанесения. От 1 штуки до тысяч единиц. Производство в России. Сроки от 3 дней.» |
| **Чекмарки (3 шт.)** | ✓ Тираж от 1 шт. · ✓ Образец перед тиражом · ✓ Быстрый ответ на заявку *(конкретный SLA — уточнить у заказчика)* |
| **CTA primary** | «Получить расчёт бесплатно» (акцентный цвет) |
| **CTA secondary** | «Смотреть каталог» (ghost-кнопка) |
| **Под CTA** | Рейтинг: Яндекс.Карты ★★★★★ (N отзывов) · 2ГИС ★★★★★ |
| **Визуал** | Реальные фото готового мерча (не рендеры) или 3 панели: B2C / WB / Корпоратив |

### 3 варианта заголовка H1 (на A/B-тест)

| # | Заголовок | Целевой уровень |
|---|-----------|----------------|
| 1 | **Мерч без сюрпризов — от идеи до готовой вещи** | Уровень 2–3, универсальный |
| 2 | **Производство мерча: DTF, шелкография, вышивка от 1 штуки** | Уровень 3–4, SEO-ориентированный |
| 3 | **Мерч, которым гордятся — для команд, маркетплейсов и компаний** | Уровень 2–3, эмоциональный |

### Структура 3 CTA-кнопок по сегментам (блок «Кто вы?»)

| Кнопка | Текст | Куда ведёт |
|--------|-------|-----------|
| Кнопка 1 | «Мерч для себя / команды» | `/` (B2C секция) или `/catalog` |
| Кнопка 2 | «Производство для WB и Ozon» | `/marketplejsy` |
| Кнопка 3 | «Корпоративный заказ» | `/korporativnym-klientam` |

## 7. TOP-20 ОБЯЗАТЕЛЬНЫХ ЭЛЕМЕНТОВ САЙТА

| # | Элемент | Почему (исследование) | Где разместить |
|---|---------|----------------------|----------------|
| 1 | **Минимальный тираж «от N штук» без скролла** | Психографика ICP: для B2C это вопрос №1 при попадании на сайт — уходят, если не нашли | Hero, карточки товаров, страница услуг |
| 2 | **3 CTA-кнопки по сегментам (Кто вы?)** | Конкурентный анализ: рынок фрагментирован, каждый сегмент требует своего пути | Hero, выше первого скролла |
| 3 | **Реальные фото готовых изделий (не рендеры)** | Психографика: «сравнивал 5 компаний — выбрал у кого больше реальных фото»; UX-бенчмарк | Hero, портфолио, карточки товаров |
| 4 | **WhatsApp + Telegram sticky-кнопки** | Психографика: B2C и WB-продавцы предпочитают мессенджеры; UX-бенчмарк (benchmerch.com pattern) | Правый нижний угол, sticky на всех страницах |
| 5 | **Мобильный sticky bottom bar (WA + Позвонить + Каталог)** | UX-бенчмарк: 78% россиян покупают с мобильных (Data Insight / АКИТ, 2024); bottom bar критичен для РУ-рынка | Мобильная версия, все страницы |
| 6 | **Калькулятор стоимости / ориентировочные цены** | Конкурентный анализ: все B2B прячут цены — прозрачность = конверсия; Шварц ур.3 | Страница услуг, каталог, WB-лендинг |

## 10. ВОПРОСЫ К КЛИЕНТУ (для Филиппа)

Только 7 блокирующих вопросов — без ответов невозможно начать разработку.

| # | Вопрос | Почему блокирует |
|---|--------|-----------------|
| **1** | **Next.js или Astro?** Если нужна CMS с динамическим каталогом и ISR — Next.js. Если сайт преимущественно статический и скорость в приоритете — Astro. Рекомендация: Next.js. | Принципиально разная архитектура рендеринга, роутинга и деплоя |
| **2** | **Нужна ли CMS / кто будет обновлять сайт?** Если заказчик сам добавляет товары, кейсы, статьи — нужен headless CMS (Sanity или Strapi). Если контент статичен при запуске — JSON/MDX. | Без ответа невозможно выбрать источник данных для каталога и блога |
| **3** | **Есть ли брендбук?** Логотип, 2–3 фирменных цвета, шрифт. | Без этого невозможно начать дизайн даже wireframe-уровня |
| **4** | **Есть ли готовые макеты в Figma?** Если нет — что значит «прототип за 4–5 дней»: wireframes или дизайн-макеты? | Определяет что именно делать на первом этапе |
| **5** | **Хостинг: Vercel или VPS?** Vercel — бесплатный план, идеален для Next.js, CI/CD из коробки. VPS — нужна ручная настройка. | Влияет на деплой, CI/CD, стоимость |
| **6** | **Telegram-бот готов?** Нужен token бота и chat_id чата/группы для приёма заявок. | Без этого форма на сайте работать не будет |
| **7** | **Кто и к какому сроку предоставляет контент?** Тексты страниц, фото продукции, логотипы клиентов, кейсы, отзывы. | Ожидание контента — главная причина срыва сроков |

4. Каталог с фильтрами
