# Next.js 15 vs Astro 5: Выбор стека для мерч-сайта

**Дата:** 19 марта 2026
**Вердикт:** Next.js 15 (App Router)

---

## Сравнительная таблица по 6 критериям

| Критерий | Next.js 15 (App Router) | Astro 5 | Победитель |
|---|---|---|---|
| **1. SEO из коробки** | Metadata API, generateStaticParams, next-sitemap, JSON-LD в Server Components | @astrojs/sitemap, Content Collections, SSG по умолчанию | Ничья |
| **2. Скорость разработки** | shadcn/ui нативно для React, богатая экосистема, больше шаблонов | shadcn/ui через @astrojs/react, дополнительный слой | **Next.js** |
| **3. Tailwind + shadcn/ui** | Нативная поддержка, все компоненты React-first | shadcn/ui требует react-интеграции, трения с hydration | **Next.js** |
| **4. Core Web Vitals** | Server Components = ноль JS по умолчанию, требует дисциплины | Zero-JS по умолчанию, Islands Architecture | Astro |
| **5. Деплой на Vercel** | Первоклассная поддержка (та же компания), нулевая конфигурация | @astrojs/vercel адаптер, работает отлично | **Next.js** (marginal) |
| **6. Headless CMS в будущем** | Sanity (next-sanity), Tina CMS — официальная поддержка | Sanity через JS SDK, Tina работает, меньше шаблонов | **Next.js** |

## Ключевые аргументы для ЭТОГО проекта

### Почему Next.js

**Скорость разработки (критично при сроке 4-5 дней на прототип):**

| Задача | Next.js | Astro | Разница |
|---|---|---|---|
| Фильтрация каталога + портфолио | ~2-3 ч | ~4-6 ч | -2-3 часа |
| Форма → Telegram (Server Action) | ~30 мин | ~1-1.5 ч | -1 час |
| Настройка shadcn/ui | ~15 мин | ~45 мин | -30 мин |

- `npx shadcn@latest init` — нативно, все компоненты без дополнительных настроек
- Фильтрация каталога: Client Components с `useState`, без гимнастики с Astro Islands
- Форма → Telegram: Server Action в том же файле, без отдельного API-роута
- 95% гайдов по shadcn/ui написаны под Next.js App Router

### Когда Astro был бы лучше
- Если бы не использовался shadcn/ui
- Если фильтрация отсутствует или тривиальна
- Если сроки не критичны
- Если сайт полностью статичный без интерактивных компонентов

## SSG/SSR/ISR стратегия для каждого типа страниц

| Страница | Стратегия | Реализация | Обоснование |
|---|---|---|---|
| Главная | SSG | `export const dynamic = 'force-static'` | Контент не меняется, максимум скорости |
| Каталог (список категорий) | SSG | Дефолт App Router | Статичный JSON-источник |
| Категория `/catalog/[slug]` | SSG | `generateStaticParams()` из JSON | Все категории известны на build-time |
| Технологии `/tech/[slug]` | SSG | `generateStaticParams()` из MDX | 6 страниц, полностью статичные |
| WB/Ozon страница | SSG | Дефолт | SEO-приоритет, максимум Core Web Vitals |
| Портфолио | SSG + Client | SSG + `"use client"` фильтр | Данные статичны, фильтрация на клиенте |
| Кейс `/cases/[slug]` | SSG | `generateStaticParams()` из MDX | MDX-файлы известны на build-time |
| Блог (список) | SSG | Дефолт | Один маршрут, статичный |
| Статья `/blog/[slug]` | SSG | `generateStaticParams()` из MDX | MDX-файлы известны на build-time |
| О компании | SSG | Дефолт | Чистый контент |
| Контакты | SSG + Client | SSG + `"use client"` для Яндекс.Карт | Карта требует браузерного API |
| Юридические | SSG | Дефолт | Редко меняются |

**ISR не нужен на старте** — контент из MDX/JSON, меняется только при деплое. ISR понадобится при подключении Sanity CMS (вебхук → revalidation).

**`output: 'export'` НЕ устанавливать** — это отключит API Routes (нужны для Server Actions форм).

## Конкретные пакеты

### MDX и контент
```bash
# Рекомендуется: contentlayer2 (типизированные Content Collections для Next.js)
contentlayer2
next-contentlayer2

# Альтернатива: нативный @next/mdx
@next/mdx
@mdx-js/react
remark-gfm
gray-matter
```

### SEO
```bash
next-sitemap          # sitemap.xml + robots.txt одним пакетом
# robots.txt также нативно: app/robots.ts (без пакетов)
# schema.org: вручную через <script type="application/ld+json"> в Server Component
```

### Аналитика
```bash
@next/third-parties   # официальный пакет Next.js: GA4 + GTM одной строкой
# Яндекс.Метрика: next/script strategy="afterInteractive" (нет нормального пакета)
```

### Яндекс.Карты
```tsx
// Не через npm — через dynamic import
const YandexMap = dynamic(() => import('@/components/YandexMap'), { ssr: false })

// В компоненте:
<Script src="https://api-maps.yandex.ru/2.1/?apikey=...&lang=ru_RU" strategy="lazyOnload" />
```

## Итоговый стек

```
Framework:    Next.js 15 (App Router)
Язык:         TypeScript (strict mode)
Styling:      Tailwind CSS 4 + shadcn/ui
Контент:      contentlayer2 (MDX для блога и кейсов) + JSON (каталог)
SEO:          next-sitemap + Metadata API + JSON-LD вручную
Analytics:    @next/third-parties (GA4) + next/script (Яндекс.Метрика)
Карты:        Яндекс.Карты через dynamic() Client Component
Формы:        Server Actions + react-hook-form + zod
Deploy:       Vercel (SSG + Serverless для Server Actions)
CMS (будущее): Sanity + next-sanity (готово к подключению)
```
