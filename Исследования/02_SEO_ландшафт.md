**Дата исследования:** 19 марта 2026

## 1. Анализ поискового ландшафта

### 1.1 Общая характеристика ниши

Рынок производства мерча и кастомной печати в России характеризуется:
- **Высокой конкуренцией** в сегменте B2C (печать от 1 шт.)
- **Средней конкуренцией** в B2B/корпоративном сегменте
- **Низкой конкуренцией** в нишевых запросах (конкретные технологии печати, мерч для маркетплейсов)
- **Растущим спросом** на мерч для маркетплейсов (WB/Ozon) — относительно новый сегмент
- **Выраженной сезонностью** — пики перед новогодними праздниками, 23 февраля, 8 марта, началом учебного года

## 4. Стратегия внутренней перелинковки

### 4.1 Принципы структуры ссылок

```
Главная
├── Каталог
│   ├── Футболки ←→ DTF, Шелкография, Сублимация, Вышивка
│   ├── Худи ←→ DTF, Шелкография, Вышивка
│   ├── Кружки ←→ Сублимация, УФ-печать, Тампопечать
│   ├── Шоперы ←→ DTF, Шелкография
│   ├── Браслеты ←→ Тампопечать, УФ-печать
│   └── Аксессуары ←→ Тампопечать, УФ-печать, Сублимация
├── Услуги принта
│   ├── DTF ←→ Футболки, Худи, Шоперы
│   ├── Шелкография ←→ Футболки, Худи, Шоперы
│   ├── Сублимация ←→ Кружки, Футболки
│   ├── УФ-печать ←→ Аксессуары, Кружки
│   ├── Вышивка ←→ Худи, Футболки
│   └── Тампопечать ←→ Кружки, Аксессуары, Браслеты
├── Маркетплейсы ←→ Каталог (все), Корпоративным клиентам
├── Корпоративным клиентам ←→ Каталог (все), Услуги (все)
├── Портфолио ←→ Каталог (релевантные), Услуги (релевантные)
├── О компании ←→ Портфолио
└── Блог ←→ Услуги (тематические), Каталог (тематические)
```

### 4.2 Конкретные рекомендации по перелинковке

1. **Продукт → Технология.** На каждой странице продукта (футболки, худи и др.) указывать доступные технологии печати со ссылками на соответствующие страницы услуг. Пример: на странице «Футболки» — блок «Доступные технологии: [DTF-печать](/uslugi/dtf-pechat), [Шелкография](/uslugi/shelkografiya), [Сублимация](/uslugi/sublimaciya), [Вышивка](/uslugi/vyshivka)».

2. **Технология → Продукт.** На каждой странице технологии — раздел «На чем можно печатать» со ссылками на продуктовые страницы.

3. **Корпоративным клиентам → Продукты + Технологии.** Страница корпоративных клиентов ссылается на конкретные продукты и технологии, подходящие для корпоративного сегмента.

4. **Маркетплейсы → Продукты.** Страница маркетплейсов ссылается на продукты, востребованные для WB/Ozon.

5. **Блог → Коммерческие страницы.** Каждая статья блога содержит CTA со ссылками на релевантные страницы продуктов/услуг. Пример: статья «DTF vs шелкография» ссылается на `/uslugi/dtf-pechat` и `/uslugi/shelkografiya`.

6. **Портфолио → Продукт + Технология.** Каждый кейс в портфолио сопровождается ссылками на использованную технологию и тип продукции.

7. **Хлебные крошки** на каждой странице — обязательный элемент навигации и перелинковки.

8. **Контекстные ссылки** в текстовом контенте (не только навигационные блоки).

9. **Блок «Рекомендуемые услуги»** или «Смотрите также» внизу каждой страницы с 3-4 релевантными ссылками.

10. **Якорные тексты** — использовать целевые ключевые слова в тексте ссылок, избегая «нажмите здесь» и «подробнее».

## 6. Технические SEO-рекомендации для Next.js / Astro

### 6.1 Рендеринг и архитектура

| Рекомендация | Описание |
|-------------|----------|
| **SSG для контента** | Блог, страницы услуг, «О компании», портфолио — генерировать статически (SSG). Astro по умолчанию работает в SSG-режиме и дает на 40% быстрее загрузку и на 90% меньше JS, чем Next.js |
| **SSR/ISR для динамики** | Каталог с фильтрами, калькулятор цен — ISR (Next.js) или SSR (Astro с `output: 'server'`). ISR позволяет кэшировать страницы и обновлять по таймеру |
| **Hybrid mode (Astro)** | Если выбран Astro — использовать гибридный режим (`output: 'hybrid'`): большинство страниц SSG, динамические — SSR |
| **Edge Functions** | Для персонализации (регион, UTM) использовать Edge Middleware, не влияя на кэширование основного HTML |

### 6.2 Метатеги и Head

```tsx
// Next.js (App Router) — layout.tsx / page.tsx
export const metadata: Metadata = {
  title: 'Печать на футболках на заказ — ...',
  description: '...',
  alternates: { canonical: 'https://example.com/catalog/futbolki' },
  openGraph: {
    title: '...', description: '...', images: ['/og/futbolki.jpg'],
    type: 'website', locale: 'ru_RU',
  },
};
```

```astro
---
// Astro — page.astro
const title = 'Печать на футболках на заказ — ...';
const description = '...';
---
<head>
  <title>{title}</title>
  <meta name="description" content={description} />
  <link rel="canonical" href="https://example.com/catalog/futbolki" />
  <meta property="og:title" content={title} />
  <meta property="og:description" content={description} />
  <meta property="og:image" content="/og/futbolki.jpg" />
  <meta property="og:locale" content="ru_RU" />
</head>
```

### 6.3 Производительность (Core Web Vitals)

| Метрика | Целевое значение | Рекомендации |
|---------|-----------------|-------------|
| **LCP** | < 2.5 сек | Оптимизация hero-изображений: WebP/AVIF, `loading="eager"`, `fetchpriority="high"`, preload. Next.js `<Image>` или Astro `<Image>` компоненты с автоматической оптимизацией |
| **FID / INP** | < 200 мс | Минимизация JS. Astro — islands architecture (гидратация только интерактивных компонентов). Next.js — `use client` только там, где нужно |
| **CLS** | < 0.1 | Указывать `width`/`height` для всех изображений. Резервирование места под шрифты (`font-display: swap`, `size-adjust`). Не вставлять контент над fold динамически |

### 6.4 Sitemap и robots.txt

```ts
// Next.js — app/sitemap.ts
export default function sitemap(): MetadataRoute.Sitemap {
  const staticPages = [
    { url: 'https://example.com', lastModified: new Date(), changeFrequency: 'weekly', priority: 1.0 },
    { url: 'https://example.com/catalog/futbolki', lastModified: new Date(), changeFrequency: 'weekly', priority: 0.9 },
    // ... все страницы
  ];
  return staticPages;
}
```

```ts
// Next.js — app/robots.ts
export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: '*', allow: '/', disallow: ['/api/', '/admin/'] },
    sitemap: 'https://example.com/sitemap.xml',
  };
}
```

### 6.5 Структурированные данные (JSON-LD)

Внедрять JSON-LD через `<script type="application/ld+json">` на каждой странице:

- **Главная:** `Organization` + `WebSite` (с `SearchAction`)
- **Каталог (категории):** `ItemList` + `Product` + `AggregateOffer`
- **Услуги:** `Service` + `Offer` + `FAQPage`
- **Блог:** `BlogPosting` + `Article`
- **Портфолио:** `ImageGallery` + `CreativeWork`
- **Контакты / О компании:** `LocalBusiness`
- **Все страницы:** `BreadcrumbList`

### 6.6 Техническая оптимизация

| Область | Рекомендация |
|---------|-------------|
| **URL-структура** | ЧПУ на транслите: `/catalog/futbolki`, `/uslugi/dtf-pechat`, `/blog/dtf-ili-shelkografiya`. Без дублей, без trailing slash (или с — но единообразно) |
| **Каноникалы** | `<link rel="canonical">` на каждой странице. Особенно важно для каталога с фильтрами/сортировками |
| **hreflang** | Не требуется (сайт только на русском) |
| **Редиректы** | 301 для всех старых URL при редизайне. Не допускать цепочек редиректов |
| **404-страница** | Кастомная 404 с поиском и ссылками на основные разделы |
| **Скорость шрифтов** | Подключать шрифты через `next/font` (Next.js) или `@fontsource` (Astro) — без внешних запросов к Google Fonts |
| **Изображения** | WebP/AVIF, lazy loading (кроме hero), alt-тексты с ключевыми словами, адаптивные `srcset` |
| **Мобильная версия** | Mobile-first дизайн, тач-удобные CTA (минимум 48x48px), нет горизонтального скролла |
| **HTTPS** | Обязательно. HSTS header |
| **Внутренний поиск** | Реализовать поиск по сайту, данные о поисковых запросах использовать для расширения семантики |

### 6.7 Индексация и краулинг

- **Astro:** Статические HTML-файлы индексируются без проблем. Убедиться, что динамические маршруты прегенерируются через `getStaticPaths()`
- **Next.js:** Для App Router — проверить, что страницы не рендерятся только на клиенте. Использовать Server Components по умолчанию
- Проверять индексацию через Google Search Console и Яндекс.Вебмастер
- Настроить Яндекс.Вебмастер (основной поиск в России): подтвердить сайт, отправить sitemap, настроить региональность
- Использовать `turbo-страницы` для Яндекса (опционально, для блога)

### 6.8 Яндекс-специфичные рекомендации

| Рекомендация | Описание |
|-------------|----------|
| **Яндекс.Вебмастер** | Подтвердить сайт, настроить региональность, мониторить ошибки индексации |
| **Яндекс.Справочник** | Добавить компанию в Яндекс.Бизнес с полными данными (адрес, телефон, фото, услуги) |
| **Турбо-страницы** | Опционально для блога — ускоренная загрузка в мобильном поиске Яндекса |
| **ИКС (Индекс качества сайта)** | Поведенческие факторы (время на сайте, глубина просмотра) критически важны для Яндекса |
| **Коммерческие факторы** | Яндекс учитывает: наличие цен, контактов, реквизитов, портфолио, отзывов, сертификатов |
| **Текстовая релевантность** | Яндекс чувствителен к переоптимизации. Тексты должны быть естественными, полезными |
