## 4. Оценка реалистичности сроков

### Объем работ (по факту ТЗ)

| Компонент | Количество |
|-----------|-----------|
| Уникальных страниц (шаблонов) | 12-14 |
| Повторяющихся блоков (компонентов) | 7 (A-G) |
| Динамических типов контента | 4 (товары, кейсы, статьи, технологии печати) |
| Форм | 3-5 (CTA, контакты, возможно на каждой странице) |
| Интеграций | 2-3 (Telegram, карты, аналитика) |

### Реалистичная оценка

| Этап | Заявленный срок | Реалистичный срок | Комментарий |
|------|----------------|-------------------|-------------|
| Прототип (wireframes/дизайн) | 4-5 дней | 4-5 дней | С AI-ассистентом и shadcn/ui — реалистично при параллельной работе. |
| Верстка + адаптация + Telegram | 5-6 дней | 5-6 дней | Компонентная система, динамические страницы, формы с Telegram, SEO-настройка. Реалистично при использовании AI + shadcn/ui. |
| Тестирование, наполнение, деплой | 3 дня | 3 дня | Кроссбраузерное тестирование, наполнение контентом, деплой на Vercel. |
| **Итого** | **12-14 дней** | **12-14 дней** | При разработке с AI-ассистентом (Claude Code). |

### Вердикт по срокам

**Сроки 12-14 дней реалистичны** при использовании AI-ассистента (Claude Code) и готового UI-kit (shadcn/ui). Ключевые условия:

1. **shadcn/ui + Tailwind CSS** — ускоряет компонентную разработку в 2-3x.
2. **Статический контент (MDX + JSON)** — без CMS на старте, подключение Sanity в будущем.
3. **AI-ассистент** — генерация компонентов, форм, SEO-настроек параллельно.
4. **Тестирование заложено** — 3 дня на финальное QA + наполнение + деплой.

### Риски по срокам

- **Ожидание контента от заказчика** — может затянуть проект. Использовать заглушки на старте.
- **Без брендбука** — возможен дополнительный цикл правок по дизайну.
- **Объём адаптивной верстки** — 12-14 шаблонов x 3 брейкпоинта — значительный объём даже с AI.
