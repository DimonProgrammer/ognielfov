## A. Blog Content Strategy

### A.1. Цель блога

Блог elfprint.ru решает три последовательные задачи:

1. **SEO-трафик.** Привлечение органического трафика по информационным запросам, где конкуренция ниже, чем по коммерческим (DTF печать что это — 3 000–5 000 показов/мес, виды печати на одежде — 2 000–4 000). Коммерческие страницы конкурируют с 20+ игроками; информационные статьи дают возможность занять топ быстрее.

2. **Осведомлённость и доверие.** Прогрев аудитории уровней 1–2 по Шварцу (в совокупности 40–65% ЦА в зависимости от сегмента). Посетитель блога — это будущий клиент, который пока не готов к заказу, но формирует ассоциацию «Огни Эльфов = эксперты в мерче».

3. **Лиды.** Каждая статья содержит 2–3 точки конверсии: внутренние ссылки на коммерческие страницы (каталог, услуги) и CTA-блоки («Рассчитать заказ», «Получить образец»). Статья работает как мост между информационным интересом и коммерческим действием.

**KPI блога:**
- Органический трафик из Яндекса и Google на /blog/ — рост на 30% за 6 месяцев
- Переходы с блога на коммерческие страницы (каталог, услуги, форма заявки) — 8–15% от визитов блога
- Позиции в топ-10 Яндекса по целевым информационным запросам — 10+ статей за 6 месяцев

### A.2. Целевые уровни осведомлённости по Шварцу

| Уровень Шварца | Доля ЦА | Роль блога | Примеры тем |
|---|---|---|---|
| **1. Неосведомлённые** (30–40%) | Самый большой пул | Вдохновить, показать что мерч существует как инструмент | «Зачем компании мерч», «Идеи подарков сотрудникам» |
| **2. Осознают проблему** (20–25%) | Основная аудитория блога | Обучить, объяснить процесс, снять страхи | «Как заказать мерч для конференции», «Welcome pack что положить» |
| **3. Осознают решение** (20–25%) | Мост к коммерческим страницам | Помочь выбрать технологию, сравнить варианты | «DTF vs шелкография», «Виды печати на одежде» |
| 4–5 | Малая доля | Блог не основной инструмент | Обслуживается коммерческими страницами, кейсами, портфолио |

**Вывод:** Блог работает преимущественно на уровни 1–3. Это генератор новой аудитории, а не инструмент закрытия сделок.

### A.3. Контентные столпы (Content Pillars)

Пять основных тематических направлений, которые покрывают все сегменты ЦА и этапы воронки:

**Столп 1: Технологии печати**
- Объяснение технологий (DTF, шелкография, сублимация, вышивка, термоперенос)
- Сравнения технологий между собой
- Практические вопросы (стойкость, уход, подготовка макетов)
- Целевая аудитория: все сегменты (B2C, WB, Corp)
- Воронка: TOFU → MOFU

**Столп 2: Корпоративный мерч и HR**
- Welcome pack, онбординг, employer branding
- Мерч для мероприятий (конференции, выставки, корпоративы)
- ROI корпоративного мерча, кейсы
- Целевая аудитория: Corp (HR, маркетологи)
- Воронка: TOFU → MOFU → BOFU

**Столп 3: Идеи и вдохновение**
- Подборки идей мерча (по отраслям, по поводам, по бюджету)
- Тренды мерча текущего года
- Лукбуки, визуальные референсы
- Целевая аудитория: все сегменты
- Воронка: TOFU

**Столп 4: Практические гайды**
- Как заказать мерч (пошаговый процесс)
- Как выбрать ткань, цвет, размерную сетку
- Как подготовить макет для печати
- Бюджетирование мерч-проектов
- Целевая аудитория: B2C, Corp (уровень 2)
- Воронка: MOFU

**Столп 5: Кейсы и производственные истории**
- Реальные проекты с деталями (тираж, технология, сроки)
- Закулисье производства
- Разборы типичных ошибок при заказе мерча
- Целевая аудитория: Corp, WB (уровни 3–4)
- Воронка: MOFU → BOFU

## D. Технический SEO-чеклист для блога

### D.1. URL-структура

```
/blog/                          → листинг статей (пагинация, фильтры по темам)
/blog/[slug]                    → отдельная статья
```

**Правила формирования slug:**
- Транслитерация (латиница), дефисы между словами
- Без дат в URL (статья обновляется — URL не меняется)
- Максимум 3–5 слов: `vidy-pechati-na-odezhde`, `welcome-pack-chto-polozhit`
- Содержит primary keyword

**Примеры:**
```
/blog/vidy-pechati-na-odezhde
/blog/dtf-pechat-chto-eto
/blog/dtf-vs-shelkografiya
/blog/welcome-pack-chto-polozhit
/blog/trendy-mercha-2026
/blog/keys-merch-dlya-konferencii
```

### D.2. Breadcrumbs Schema

Каждая статья блога должна иметь хлебные крошки с разметкой `BreadcrumbList`:

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Главная",
      "item": "https://elfprint.ru"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Блог",
      "item": "https://elfprint.ru/blog"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Виды печати на одежде",
      "item": "https://elfprint.ru/blog/vidy-pechati-na-odezhde"
    }
  ]
}
```

Яндекс поддерживает BreadcrumbList и отображает хлебные крошки в сниппете.

### D.3. Article Schema.org

Каждая статья размечается как `Article` (или `BlogPosting`):

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Виды печати на одежде: 6 технологий — когда какую выбрать",
  "description": "Разбираем 6 технологий нанесения на одежду...",
  "image": "https://elfprint.ru/images/blog/vidy-pechati-og.jpg",
  "datePublished": "2026-04-01",
  "dateModified": "2026-04-01",
  "author": {
    "@type": "Organization",
    "name": "Огни Эльфов",
    "url": "https://elfprint.ru"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Огни Эльфов",
    "logo": {
      "@type": "ImageObject",
      "url": "https://elfprint.ru/images/logo.svg"
    }
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://elfprint.ru/blog/vidy-pechati-na-odezhde"
  }
}
```

### D.4. Author Markup

Для блога «Огни Эльфов» автором выступает организация, а не конкретный человек (пока нет публичных экспертов). При появлении авторов-экспертов:

```json
"author": {
  "@type": "Person",
  "name": "Филипп [Фамилия]",
  "jobTitle": "Основатель Огни Эльфов",
  "url": "https://elfprint.ru/o-kompanii"
}
```

**Для Яндекса:** зарегистрировать все тексты в разделе «Оригинальные тексты» Яндекс.Вебмастера ДО публикации — это сигнал авторства.

### D.5. Table of Contents (для длинных статей)

Для статей длиннее 2 000 слов — автоматическое оглавление (TOC):

- Генерировать из H2-заголовков
- Размещать после первого абзаца (до первого H2)
- Кликабельные якорные ссылки (`#dtf-pechat`, `#shelkografiya`)
- На мобильных — сворачиваемый блок

**Техническая реализация (Next.js):**
- Парсить заголовки из MDX/контента
- Генерировать `id` для каждого H2
- Рендерить список ссылок в компоненте `<TableOfContents />`

### D.6. Правила внутренней перелинковки

1. **Каждая статья блога ссылается минимум на 2 коммерческие страницы** (каталог или услуги). Это основная задача блога — перенаправлять SEO-трафик на конверсионные страницы.

2. **Каждая статья ссылается на 1–2 другие статьи блога** для формирования контентного кластера.

3. **Коммерческие страницы ссылаются обратно на статьи блога** — в блоке «Полезные материалы» или «Узнайте больше».

4. **Анкоры (текст ссылок) содержат ключевые слова**, но выглядят естественно:
   - Хорошо: «Подробнее о [DTF-печати](/uslugi/dtf-pechat)»
   - Плохо: «Подробнее [здесь](/uslugi/dtf-pechat)»

5. **Кластерная перелинковка:**
   ```
   Столп «Технологии» (кластер):
   /blog/vidy-pechati-na-odezhde (pillar)
     ↔ /blog/dtf-pechat-chto-eto
     ↔ /blog/dtf-vs-shelkografiya
     ↔ /blog/sublimaciya-chto-eto
     ↔ /blog/vyshivka-ili-pechat

   Столп «Корпоративный мерч» (кластер):
   /blog/idei-korporativnogo-mercha (pillar)
     ↔ /blog/welcome-pack-chto-polozhit
     ↔ /blog/merch-dlya-konferencii
     ↔ /blog/trendy-mercha-2026
     ↔ /blog/merch-hr-brend
   ```

### D.7. Оптимизация изображений

1. **Формат:** WebP (основной) + JPEG fallback. Next.js `next/image` обрабатывает автоматически.

2. **Alt-теги:** содержат LSI-ключи, описывают изображение:
   - Хорошо: `alt="DTF-печать на чёрной футболке — пример полноцветного принта"`
   - Плохо: `alt="фото"` или `alt="image1"`

3. **Размеры:** для OG-изображений — 1200x630px. Для контентных — максимум 1200px по ширине.

4. **Lazy loading:** все изображения ниже первого экрана — `loading="lazy"`.

5. **Имена файлов:** транслитерация с ключами: `dtf-pechat-na-futbolke.webp`, не `IMG_2847.jpg`.

6. **Сжатие:** целевой размер файла — до 100 KB для контентных изображений, до 200 KB для hero-изображений.

### D.8. FAQ Schema для статей с вопросами

Статьи, содержащие блок FAQ, размечаются дополнительно как `FAQPage`:

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Какая печать самая стойкая?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Шелкография и вышивка — самые стойкие технологии. DTF-печать выдерживает 50–60 стирок при правильном уходе."
      }
    },
    {
      "@type": "Question",
      "name": "Можно ли печатать на тёмной ткани?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Да. DTF-печать и шелкография подходят для тёмных тканей. Сублимация работает только на белой или светлой синтетике."
      }
    }
  ]
}
```

Яндекс поддерживает FAQPage и может отображать вопросы-ответы в расширенном сниппете, что увеличивает CTR.

### D.9. Чеклист перед публикацией статьи

```
□ URL содержит primary keyword (транслитерация, дефисы)
□ Meta title — до 60 символов, содержит primary keyword и «| Огни Эльфов»
□ Meta description — 150–160 символов, содержит keyword + CTA
□ H1 — уникальный, содержит primary keyword, отличается от meta title
□ Primary keyword — в первом абзаце текста
□ Все H2 содержат LSI/secondary keywords
□ Alt-теги заполнены для всех изображений
□ Минимум 2 внутренние ссылки на коммерческие страницы
□ Минимум 1 внутренняя ссылка на другую статью блога
□ CTA-блоки размещены (после ключевого раздела + в конце)
□ FAQ-блок есть и размечен FAQPage schema
□ Article schema.org добавлена
□ BreadcrumbList schema добавлена
□ OG-изображение (1200x630) подготовлено
□ Текст зарегистрирован в «Оригинальных текстах» Яндекс.Вебмастера
□ Текст проверен по анти-ИИ чеклисту (copywriting-guide.md)
□ Canonical URL указывает на себя
□ Статья добавлена в sitemap.xml
```
